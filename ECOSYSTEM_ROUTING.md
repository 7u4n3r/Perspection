# Ecosystem Routing Map

This document defines the operational routing between L900, Perspection, Trinity, DisLexicon, Faceless Reclaimation, and related project surfaces.

The purpose is to prevent identity bleed, project collapse, and unnecessary merging between layers.

Interoperable does not mean merged.

---

# Core routing principle

Each layer has one primary function.

No layer should absorb the purpose of another layer.

When a piece of content, tool, page, product, or service is unclear, route it by function instead of emotional intensity.

---

# Layer definitions

## L900

Primary role: parent infrastructure.

L900 is the container, studio, publishing mark, operating infrastructure, and routing layer behind the work.

L900 is not the public explanation for every project.

Belongs here:

- umbrella identity
- publishing infrastructure
- project routing
- parent archive
- operational records
- cross-project index
- monetization container
- studio-level positioning

Does not belong here:

- deep testimony as primary content
- full DisLexicon entries
- Perspection methodology in full
- Trinity instructional content in full
- emotional processing material

---

## Perspection

Primary role: orientation and discernment practice.

Perspection provides active lens provision for discernment development.

Belongs here:

- orientation principles
- discernment language
- session methodology
- service definitions
- constitutional boundaries
- governance documents
- observational restraints
- frame and pattern visibility

Does not belong here:

- full product catalog
- personal testimony as the main body
- recovery-branded material
- every framework in detail
- sales-heavy language
- algorithmic content strategy

---

## Trinity Framework

Primary role: production logic.

Trinity defines how one source can move through two outputs into one motion.

Belongs here:

- one source, two outputs, one motion
- production mapping
- content transformation logic
- artifact generation logic
- workflow simplification
- project motion tracking

Does not belong here:

- identity claims
- Perspection service positioning
- DisLexicon definitions in full
- governance doctrine
- public-facing emotional testimony

---

## DisLexicon

Primary role: language infrastructure.

DisLexicon names terms, distinctions, and state-aware language used across the ecosystem.

Belongs here:

- canonical terms
- definitions
- second-person language
- language Reclaimation
- term usage rules
- approved spelling and vocabulary
- language audits

Does not belong here:

- business model as primary function
- session structure as primary function
- full personal memoir
- project governance beyond language rules

---

## Faceless Reclaimation

Primary role: applied lived navigation.

Faceless Reclaimation is where the work becomes usable for people navigating pressure, identity strain, recovery language, visibility, survival, and self-directed rebuilding.

Belongs here:

- guides
- zines
- lived navigation tools
- Reclaimation language
- harm-reductive support material
- faceless digital products
- public-facing applications
- field-tested practices

Does not belong here:

- Perspection constitutional governance in full
- L900 umbrella doctrine
- Trinity as identity
- unrelated product sprawl
- every archive fragment

---

## Pattern Library

Primary role: pattern documentation.

The Pattern Library stores observed patterns without forcing every pattern into a service, product, or identity claim.

Belongs here:

- pattern names
- pattern descriptions
- before/after visibility examples
- observed mechanisms
- application notes
- examples across domains

Does not belong here:

- personal branding
- therapy-style processing
- broad ideology
- all project pages

---

# Routing questions

When placing any item, ask:

1. Is this orientation or discernment?  
Route to Perspection.

2. Is this production logic or transformation workflow?  
Route to Trinity.

3. Is this terminology or language correction?  
Route to DisLexicon.

4. Is this applied support for lived navigation?  
Route to Faceless Reclaimation.

5. Is this parent structure, publishing, or cross-project organization?  
Route to L900.

6. Is this an observed recurring mechanism?  
Route to Pattern Library.

---

# Hard refusals

Perspection does not become Faceless Reclaimation.

Faceless Reclaimation does not become Perspection governance.

Trinity does not become identity.

DisLexicon does not become the whole business.

L900 does not need to explain everything publicly.

Pattern documentation does not automatically become service language.

Interoperable does not mean merged.

---

# Operational use

This document should be used when:

- rebuilding websites
- updating GitHub repos
- instructing Manus
- sorting Google Drive materials
- deciding where new pages belong
- separating public-facing copy from internal doctrine
- preventing project collapse under pressure

If a project starts absorbing too many functions, return to this routing map before adding more content.
