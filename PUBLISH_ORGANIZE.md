# PUBLISH_ORGANIZE.md

## Purpose

This file defines the publish/organize posture for `Perspection`.

Perspection is the governance and observation infrastructure spine of the ecosystem.

It may be published as a reference surface.
It should not be published as a healing journey, story-reclaiming project, Trinity method page, or product funnel.

## Current public role

Perspection is a reference surface for:

- System Refusal governance
- observation-only interface constraints
- copy lint rules
- audit workflows
- non-coercive orientation infrastructure
- code-reference material related to those constraints

## Publish-safe description

Perspection is governance and reference infrastructure for observation-only systems.

It holds refusal rules, copy constraints, audits, and implementation references for systems that preserve orientation without forcing motion.

## What this is

Perspection may hold:

- governance stack documents
- copy lint rules
- audit workflows
- refusal constraints
- observation-only standards
- reference implementations
- evidence/structure material related to governance behavior

## What this is not

Perspection is not:

- Faceless Reclaimation
- the Trinity Framework site
- the Living Survival Guide
- the ABLE-KY county map
- a product funnel
- a recovery sequence
- a testimony archive
- a public story body

## Entry posture

The Perspection entry surface should be:

- quiet
- observational
- non-sequential
- non-optimizing
- non-coercive
- free from urgency language
- free from progress framing
- free from obligation language
