## Fast routing

### Put material in `Perspection` when it is:

- governance
- refusal infrastructure
- observation-only constraint
- copy lint
- audit workflow
- code-reference for non-coercive orientation infrastructure

### Put material in `faceless-reclaimation` when it is:

- public Reclaimation material
- lived-language artifact
- zine/product/landing page copy
- story reclaiming
- public-facing fire language

### Put material in `trinity-framework-site` when it is:

- Trinity-specific explanation
- Encounter / Interrogation / Integration
- framework diagrams
- method structure
- public Trinity site copy

### Put material in `Trinity-tools` when it is:

- Manus export
- UI experiment
- unsorted tool build
- shared prototype
- material awaiting classification

### Put material in `system-analysis-reference` when it is:

- evidence organization template
- timeline structure
- claim/source/interpretation separator
- neutral case-study pattern
- institutional or procedural analysis format

### Put material in `ky-county-map` when it is:

- Kentucky county navigation
- ABLE-KY resource routing
- civic/legal-access county tool
- static map improvement

## Repair order

When the system feels messy, repair in this order:

1. Stop adding features.
2. Identify which repo the material belongs in.
3. Preserve the source before rewriting.
4. Classify the source.
5. Move or copy only after the destination is clear.
6. Update the relevant README or boundary file if needed.
7. Log the change in an issue or checkpoint file.

## Manus instruction

When using Manus, start with diagnosis.

Do not let Manus redesign, merge identities, delete preserved files, or rewrite public copy until it has read the relevant boundary files.

Use:

`7u4n3r/Trinity-tools/docs/prompts/MANUS_RECOVERY_PROMPT.md`

when the build or preview becomes unstable.

## Stop condition

If the question is “where does this go?” do not publish it yet.

Route first.
Then rewrite.
Then deploy.
