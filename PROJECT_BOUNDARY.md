# PROJECT_BOUNDARY.md

## Project identity

This repository is the canonical Perspection governance and reference surface.

It holds System Refusal governance documents, observation-only constraints, copy lint rules, audits, and code-reference material for non-coercive orientation infrastructure.

## This project is not

- It is not Faceless Reclaimation.
- It is not the Living Survival Guide.
- It is not the Trinity product/site/app container.
- It is not a healing journey, coaching sequence, method funnel, or personal testimony archive.

## Allowed material

- Governance stack documents
- System refusal rules
- Copy lint and audit workflows
- Observation-only interface constraints
- Evidence/structure/reference material
- Code-reference examples that demonstrate governance constraints

## Not allowed material

- Instructional recovery copy
- Personal narrative as primary content
- Product funnel language
- Obligation or progress framing
- Claims that the user is late, behind, wrong, incomplete, or in need of a next step
- Ecosystem copy that merges Perspection with Faceless Reclaimation, Trinity, or Living Survival Guide

## Related but separate

- `Trinity-tools` may hold working tool builds and implementation experiments.
- `faceless-reclaimation` may hold Reclaimation-facing public materials.
- `trinity-framework-site` may hold Trinity-specific site material.
- `system-analysis-reference` may hold neutral analysis/reference material.

## Operating rule

Perspection observes structure.

It does not instruct recovery.
It does not request motion.
It does not become testimony.
