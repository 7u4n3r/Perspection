# ECOSYSTEM_MAP.md

## Purpose

This file maps the current project ecosystem so related repositories can remain interoperable without merging identities.

It is a routing reference, not a hierarchy of importance.

## Operating rule

Interoperable does not mean merged.

No project may redefine another project's purpose.

## Routing questions

When adding new material, ask:

1. Is this governance/refusal infrastructure?  
   Route to `Perspection`.

2. Is this public-facing lived Reclaimation material?  
   Route to `faceless-reclaimation`.

3. Is this Trinity-specific method explanation or site structure?  
   Route to `trinity-framework-site`.

4. Is this an experiment, Manus export, or unsorted tool build?  
   Route to `Trinity-tools` until sorted.

5. Is this neutral evidence organization, timeline, or claim-boundary structure?  
   Route to `system-analysis-reference`.

6. Is this Kentucky county navigation or ABLE-KY resource routing?  
   Route to `ky-county-map`.

## Anti-merge checks

A file should be moved or rewritten if it does any of the following:

- Makes Perspection sound like a healing journey, method funnel, or product.
- Makes Trinity sound like a personal identity or doctrine.
- Makes Faceless Reclaimation sound like governance infrastructure.
- Makes the Living Survival Guide absorb unrelated ecosystem projects.
- Treats ABLE-KY navigation as a general legal archive.
- Mixes claim, source, and interpretation without separation.
